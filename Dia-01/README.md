## Dia 01: O Marco Inicial e a Conexão com a Nuvem

### Prefácio:

O **Dia 01** marca o nascimento do projeto **Noach**. Este dia foi dedicado à construção da base de tudo: a preparação do ambiente de desenvolvimento. Sair do zero absoluto exige configurar as ferramentas que permitem ao computador entender nossas instruções. Focamos em transformar uma máquina comum em uma estação de trabalho de um desenvolvedor, estabelecendo a ponte entre o código local e o armazenamento global via GitHub.

### 1. Preparação do Motor e do Editor:

Para programar, precisamos de duas coisas essenciais: um motor para interpretar o código e um editor para escrevê-lo. Instalamos o **Python**, que é a linguagem que dará vida às nossas ideias, e o **VS Code**, o editor profissional onde passaremos a maior parte do tempo organizando nossos pensamentos em linhas de código.

````
# Exemplo 1: O primeiro contato com a sintaxe Python
# A função print exibe informações na tela do console
print("Olá mundo! Eu sou o Noach e este é meu primeiro código.")
````

### 2. Configuração de Identidade (Git):

Um programador precisa assinar seu trabalho. Aprendemos que o **Git** não é apenas uma ferramenta de cópia, mas um sistema que registra quem fez o quê. Configuramos a identidade global para que cada evolução no código seja atribuída corretamente ao seu autor.

````
# Exemplo 2: Comandos de configuração de identidade no terminal
git config --global user.name "Seu Nome"
git config --global user.email "seu-email@exemplo.com"
````

### 3. O Ciclo de Sincronização:

A grande conquista do dia foi entender como tirar o código do computador físico e enviá-lo para o **GitHub**. Esse processo protege os estudos contra perdas e permite que o progresso seja acompanhado de qualquer lugar. O ciclo de "Adicionar, Comitar e Enviar" (Add, Commit, Push) tornou-se nosso primeiro ritual diário.


### 4. Estrutura de Pastas e Organização:

Além do código, aprendemos a importância de manter a "casa" organizada. Criar pastas específicas para cada dia de estudo facilita a revisão futura e demonstra disciplina, uma característica vital para qualquer desenvolvedor que pretenda construir sistemas complexos.

````
# Exemplo 4: Estrutura inicial do repositório
/Noach-Python-SQL
    /Dia-01
        hello_world.py
    README.md
````

### Observações Finais:

O Dia 01 foi curto em linhas de código, mas imenso em infraestrutura. O "Hello World" não é apenas uma tradição, é o teste final que confirma que o Python, o Editor e o Terminal estão conversando perfeitamente. Com a conexão estabelecida com o GitHub, o projeto Noach agora está oficialmente documentado na nuvem, pronto para as complexidades de SQL e APIs que virão nos próximos 50 dias.
